# AtomPump Apps Docs Page

## Summary

Added a local AtomPump documentation page at `/docs/apps` for customers who need
to connect AI Agent clients through the OpenAI-compatible AtomPump API.

## Scope

- Backend code was not changed.
- UI implementation stayed under `web/atompump/`.
- Shared workflow notes were written under `docs/gstack/`.

## Changes

- Added a public `/docs/apps` route.
- Added a lightweight docs page covering quick start, connection fields, common
  AI Agent clients, protocol paths, CLI/coding agent setup, generic
  OpenAI-compatible config, and troubleshooting.
- Changed public `Docs` navigation from the external `docs.newapi.pro` link to
  the local `/docs/apps` page.
- Updated `/explore` custom navigation to use the local docs route.
- Added zh/en translations for the docs page. Other locales use English
  fallback entries generated by the existing i18n sync workflow.
- Added CLI/coding agent examples for Codex CLI, Claude Code / Claude CLI,
  Factory Droid CLI, OpenClaw, LangBot, and AstrBot.
